# Plan: Add job posting date (`posted_at`) + restore source/recruiter info

## Goals
1. Show users when a job was actually posted, not when we crawled it
2. Restore recruiter/aggregator indicator and source info on job cards (dropped during redesign)

## Current state
- `RawJob` has no `posted_at` field
- DB `jobs` table has `first_seen` / `last_seen` (crawl timestamps only)
- HN API returns `created_at_i` (unix epoch) per comment â€” this IS the posting date
- LinkedIn search HTML contains `<time class="job-search-card__listdate" datetime="...">` â€” this IS the posting date
- Greenhouse API returns `updated_at` per job â€” posting/update date
- Lever API returns `createdAt` (unix ms) per posting â€” posting date
- NextPlay already serializes `j.posted_at` to cache but `RawJob` lacks the field (latent bug)
- NextPlay cache also writes `j.salary` but `RawJob` has `salary_text` (latent bug)
- `company_intel` field exists on `JobSummary` (contains "recruiter posting" note) but JobCard doesn't render it
- `sources_seen` computed in JobCard but never displayed

## Constraints
- All collectors must produce ISO 8601 strings for `posted_at` (normalize at source)
- Don't touch `shortlist/db.py` (CLI is frozen)
- Ashby posting dates deferred (needs GraphQL schema change)

**File:** `shortlist/collectors/hn.py`
- In `_parse_single_comment()`, read `comment["created_at_i"]` (unix epoch)

- Pass as `posted_at` kwarg to `RawJob()`

**Test:** Unit test with mock comment containing `created_at_i`, assert `RawJob.posted_at` is ISO string

### 3. LinkedIn collector: extract posting date

- Pass matching date as `posted_at` to `RawJob()` (index-matched like other fields)
- If fewer dates than jobs, default missing to `None`

**Test:** Unit test with mock HTML containing `<time>` tags, assert `RawJob.posted_at` set

### 4. ATS fetchers: extract posting dates
